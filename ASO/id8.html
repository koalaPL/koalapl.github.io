ADMINISTRACJA SYSTEMAMI OPERACYJNYMI / Serwer Web www /part 2 - IIS

Stronę WwW z obsługą interpretera PHP i bazy danych MySQL

W poprzednim rozdziale zainstalowaliśmy role "Serwer sieci Web (IIS)". Jeżeli tego nie uczyniłeś, zrób to teraz.
Zanim zaczniemy konfigurację ISS musimy pobrać zasoby niezbędne do uzyskania konicowego wyniku.
Pliki strony WwW Świat koali : dostępne w zakładce Zasoby do pobrania.
Interpreter PHP: https://windows.php.net/downloads/releases/php-8.2.3-nts-Win32-vs16-x64.zip
MySQL Community: https://cdn.mysql.com//Downloads/MySQLInstaller/mysql-installer-community-8.0.32.0.msi
Microsoft Visual C++ 2015-2019 Redistributable (x64): https://download.visualstudio.microsoft.com/download/pr/b929b7fe-5c89-4553-9abe-6324631dcc3a/296F96CD102250636BCD23AB6E6CF70935337B1BBB3507FE8521D8D9CFAA932F/VC_redist.x64.exe
Microsoft .NET Framework 4.5.2 (instalator offline): https://download.microsoft.com/download/E/2/1/E21644B5-2DF2-47C2-91BD-63C560427900/NDP452-KB2901907-x86-x64-AllOS-ENU.exe

Zainstaluj Dodatek gościa i udostępnij folder Gospodarza dla WindowsServer2012R2. Umieść w nim pliki które pobrałeś w katalogu udostepnionym dla maszyny wirtualnej. Z archiwum "Pliki strony świat koali.zip" przenieś katalog "Strony" wraz z zawartością na dysk lokalny C:\. Usuń domyślną witrynę "Default Web Site" i dodaj nową wskazując na lokalizację C:\Strony, uzupełniając pozostałe lubryki według schematu.
Przenieś index.html na górę listy plików. Możesz dodać niestandardową nazwę pliku strony WwW, klikając "Dodaj..." i wpisując jej nazwę. Witryna WwW już działa, ale nie do konica. Nie ma interpretera PHP bzy danych MySQL oraz zdjecia się niewczytują
(chyba że dodasz drugą kartę sieciową NAT na kliencie Windows 10). Poprzez dodawanie ról doinstaluj Interfejs CGI: Zainstaluj na serwerze: Microsoft Visual C++ 2015-2019 Redistributable (x64). 1. Rozpakuj arhiwum "php-8.2.3-nts-Win32-vs16-x64.zip".
2. Zmień nazwę katalogu na "PHP".
3. Przenieś na Dysk lokalny C:\
Przejdź w Menadżerze internetowych usług informacyjnych (IIS) do "Mapowania obsługi".
Dodaj mapowanie modułu...
Ścieżka żądania: *.php
Moduł: FastCgiModule
Plik wykonywalny: C:\PHP\php-cgi.exe
Nazwa: Tu możesz wpisać co kolwiek, ale warto aby nazwa była smoopisująca się. Ja to mapoiwanie nazwałem PHP_FastCGI poniważ wykorzystuje moduł FastCgi do opsługi interpretera PHP. Po kliknięciu "Ograniczenia żądań..." ukaże się okno z trzema zakładkami. Poniżej przedstawiono poprawna konfigurację pod mapowanie PHP. Tu klikamy oczywiście "Tak". Jak widać poniżej Interpreter PHP działa poprawnie. Pozostała nam jeszcze obsługa bazodanowa MySQL. Zainstaluj na serwerze: Microsoft .NET Framework 4.5.2 (instalator offline). Koniecznie uruchom ponownie teraz serwer. Rozpoczynamy instalację MySQL Community (mysql-installer-community-8.0.32.0.msi).
Klikaj cały czas Next, Execute oraz Yes, aż do strony "Accounts and Roles": Podajemy hasło do root'a czyli takiego administratora
Po kliknięciu "Add User" tworzymy następującego użytkownika ( Wszelkie dane podane na obrazku poniżej): Po kolejnej serji przeklikiwania napodkasz taką strone. Musisz podać hasło do root'a, po czym kliknąć "Check".
Gdy połączenie zakoniczy sie sukcesem klikaj dalej. Po instalacji wyskoczą takie dawa okna zamknij je. Do niczego nam się nie przydadzą :p Uruchom wiersz poleceń MySQL: Podaj hasło do root'a, następnie wykonaj dwa zapytania SQL widoczne na obrazku poniżej: W katalogu C:\PHP znajduje się plik "php.ini-production" będocym templatką konfiguracji PHP.
Skopiuj go i zmień nazwę na "php.ini" Modyfikując "php.ini" wyszukaj frazę "extension=" i usuń znak średnika ";" dla rozszerzeń "mysqli" oraz "pdo_mysql". Serwer na którym wykonuje konfigurację ma ustawiony statyczny adres 192.168.0.10, dlatego muszę zmodyfikować plik "C:\Strony\indexMySQL.php". Teraz serwer Web jest wpełni skonfigurowany bo, oprócz wyświetlania stron html, interpretuje język PHP oraz posiada działającą bazę danych MySQL.
